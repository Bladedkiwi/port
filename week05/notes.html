Modules

The JavaScript Module is basically a small unit that's independent, and reusable. In other words, it's like one lego among the entire set of legos for the Millennium Falcon 


function Rectangle()

What is export, and what is its purpose?

Simply put, export allows that modules/variables/objects/etc. to be used in other programs with the usage of import. 


//file math.js 
function square(x) { 
    return x * x; 
} 
function cube(x) { 
    return x * x; 
} 
export { square, cube }; 
    
    
//while importing square function in test.js 
import { square, cube } from './math; 
console.log(square(8)) //64 
console.log(cube(8)) //512

Named Exports

  • Useful for exporting several values
  • With Import - Mandatory: Use same name of the corresponding object.

Default Exports

  • Useful for exporting only a single object, function, or variable.
  • With Import - use any name to import

What are Prototypes, are they objects or pretend objects?

JavaScript is a prototype based language - essentially meaning they are important. 


    // function constructor 
function Person(name, job, yearOfBirth){   
    this.name= name; 
    this.job= job; 
    this.yearOfBirth= yearOfBirth; 
} 
// adding calculateAge() method to the Prototype property 
Person.prototype.calculateAge= function(){  
    console.log('The current age is: '+(2019- this.yearOfBirth)); 
} 
console.log(Person.prototype); 
    
// creating Object Person1 
let Person1= new Person('Jenni', 'clerk', 1986);  
console.log(Person1) 
let Person2= new Person('Madhu', 'Developer', 1997); 
console.log(Person2) 
    
Person1.calculateAge(); 
Person2.calculateAge();

Essentially

  • The special constructor function is like the __init__ function inside classes from python because it acts like the base template for that object and initializes in a similar way.
  • Classes in JavaScript act like special functions that simply allow for easy grouping/reading
  • Every Function has the prototype property
  • Prototype property is basically an object that we attach additional methods/properties to the base Object.
  • The base object is the initial setup of the class or function.
  • Why not just alter the class instead of use Prototype? Prototype allows for altering default classes when you don't have access to that class declaration, which makes them useful in that way
  • In the declaration of calculateAge() prototype property, the Person will now always have calculate age a method attached to it. In fact, it becomes the new default.
